🎯 1. Purpose & Scope

This skill reads any named skill's SKILL.md file and maps its content into a standardised HTML SOP template, producing a fully populated, professionally formatted SOP document. It ensures every skill in Kevin's stack has consistent, readable documentation without manual HTML authoring. The output is written directly to the outputs directory, ready for upload to the SOP index on GitHub.

ℹ️
Scope: In scope: any skill located at /mnt/skills/user/[skill-name]/SKILL.md. Out of scope: public skills in /mnt/skills/public/, skills that do not yet have a SKILL.md, and any manual editing of the SOP template CSS or structure.

💡 2. When to Use This Skill

💬 Trigger Phrases

Use this skill when Kevin or the team says any of the following:

  • "generate the SOP for [skill name]"
  • "create an SOP for [skill name]"
  • "add an SOP for [skill name]"
  • "build the SOP for [skill name]"
⚠️
Do not use this skill when: the target skill does not exist at the expected path, when Kevin is asking to update an existing SOP rather than generate a new one, or when the request is for a public skill rather than a user skill.

1

Identify the Skill

manual
  1. Extract the skill name from Kevin's message (e.g. "generate the SOP for contractor-job-creator" → contractor-job-creator).
  2. Construct the expected path: /mnt/skills/user/[skill-name]/SKILL.md.
  3. If the skill name is ambiguous, confirm with Kevin before proceeding.
2

Read the SKILL.md

automated
  1. Use the view tool to read the full SKILL.md at the constructed path.
  2. If the file does not exist, stop immediately and inform Kevin — do not proceed.
  3. Read the entire file before mapping any placeholders.
ℹ️
If the file is truncated due to length, use view_range to read the remaining sections before continuing.
3

Map All Placeholders

automated
  1. Work through every {{PLACEHOLDER}} in the template using the mapping table in the SKILL.md.
  2. Base all content strictly on the SKILL.md — do not invent or assume details.
  3. Where no relevant content exists, write N/A or a factual fallback.
  4. Never leave a placeholder unfilled in the output.
  5. Generate all HTML snippet blocks (flow steps, workflow cards, inputs table rows, tools table rows, troubleshooting items) using the exact formats defined in the SKILL.md.
⚠️
Do not alter any CSS, class names, element structure, or IDs from the embedded template. Do not add elements not present in the template.
4

Write the HTML Output File

automated
  1. Use create_file or bash_tool to write the completed HTML to /mnt/user-data/outputs/[skill-name].html.
  2. Do not stream the HTML content into the chat — write it directly to the file.
  3. Use the embedded template from the SKILL.md verbatim, with all placeholders replaced.
5

Confirm to Kevin

manual
  1. Once the file is written, output exactly: ✅ SOP generated: [skill-name].html — Upload to: github.com/chaichoong/sops in the sops/ folder.
  2. No other output is required.

🚨 8. Error Handling

  • SKILL.md not found: Stop immediately and tell Kevin the file does not exist at the expected path. Do not attempt to generate an SOP with invented content.
  • SKILL.md truncated: Use view_range to read the remaining lines before proceeding with placeholder mapping.
  • Placeholder mapping ambiguity: If a section of the SKILL.md is unclear, use the most factually defensible interpretation or write N/A. Do not invent details.
  • File write failure: If create_file fails, fall back to bash_tool with a heredoc or echo approach. Inform Kevin if both methods fail.
  • Skill name ambiguous: If Kevin's message could refer to more than one skill, ask for clarification before reading any file.

🛠️ 9. Troubleshooting

❌ The output HTML still contains unfilled {{PLACEHOLDER}} tokens
✅ Re-read the SKILL.md carefully. Every placeholder must be mapped. If no content exists for a field, write N/A — never leave a token in place.
❌ The SKILL.md file is not found at the expected path
✅ Verify the skill name is spelled exactly as the folder name under /mnt/skills/user/. Check with Kevin if unsure — do not guess.
❌ The SOP renders with broken layout or missing styles
✅ Ensure the full CSS block and template structure from the SKILL.md was used verbatim. Do not modify class names, element types, or add extra wrapper elements.
❌ The file was not written to the outputs directory
✅ Use bash_tool as a fallback. Verify the path is /mnt/user-data/outputs/[skill-name].html and not the working directory.
