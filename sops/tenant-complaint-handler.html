🎯 1. Purpose & Scope

Handles inbound tenant complaints and maintenance issues by sourcing all relevant context from Airtable, Gmail, Google Drive, and GHL before drafting a professional reply. The reply is direct, firm, and fair — written as Kevin and signed off by Erica on his behalf. All replies require Kevin's approval before any action is taken.

ℹ️
Scope: In scope: tenant complaints, maintenance issues, disputes, and any written response required to a tenant. Out of scope: sending GHL/SMS messages directly (Claude cannot send from GHL), or taking legal action without Kevin's explicit instruction.

💡 2. When to Use This Skill

💬 Trigger Phrases

Use this skill when Kevin or the team says any of the following:

  • "New complaint" / "Tenant issue"
  • "Tenant is saying [X]"
  • "Problem at [property]"
  • Kevin describes a tenant situation that needs a written response
⚠️
Do not use this skill when: Routine tenant updates or rent reminders — this skill is for complaints, disputes, and issues that require a considered written response.

1

Source Context from All Systems

automated
  1. Airtable: search Tenants table by name/address. Pull tenancy record, rent status, open tasks, prior flags.
  2. Gmail (kevin@runpreneur.org.uk): search for emails from/about the tenant. Note prior complaints, commitments made.
  3. Google Drive: search for tenancy agreement, inspection reports, inventory, Schedule of Works for the property.
  4. GHL: check contact record and conversation history for the tenant.
  5. Flag if relevant context may exist in Evernote/NotebookLM/ClickUp — ask Kevin to check manually.
2

Summarise and Get Approval to Draft

approval gate
  1. Present summary: Tenant, Property, Issue, Context Found (per system), Risks/Flags.
  2. Ask: 'Happy for me to draft the reply based on this, or anything to add?'
  3. Only proceed to drafting after Kevin confirms.
3

Draft and Present Reply

approval gate
  1. Write the reply as Kevin. Tone: direct, professional, fair.
  2. Valid complaint: acknowledge, state action and timeline.
  3. Unclear/disputed: ask for more detail or clarify facts.
  4. Unreasonable: hold position firmly but respectfully.
  5. No corporate filler. Max 3–4 short paragraphs.
  6. Sign off: 'Kind regards, Erica / (sent on behalf of Kevin Brittain)'.
  7. Present draft. Ask: 'Approve to send?' / 'Any amendments?'. Do not send without confirmation.
4

Post-Approval Actions

automated
  1. Create an Airtable task using airtable-task-creator if any action is needed (repair, inspection, follow-up).
  2. Log the complaint in the tenant's Airtable record or Tasks table with today's date.
  3. Draft or send the reply: email → Gmail draft; GHL/SMS → note the message for Kevin to send from GHL.

🚨 8. Error Handling

  • Tenant not found in Airtable: Try searching by property address. Flag to Kevin if no record exists.
  • No prior correspondence in Gmail: Note absence of history. This may mean the complaint is new or came via another channel.
  • GHL not directly accessible: Note it as a gap and ask Kevin if there is relevant GHL context to incorporate.

🛠️ 9. Troubleshooting

❌ Cannot access GHL contact record
✅ Flag to Kevin that GHL context could not be checked. Kevin can provide relevant notes manually.
❌ Reply drafted but Kevin wants a different tone
✅ Amend per Kevin's instruction before creating the Gmail draft. Do not send the original version.
