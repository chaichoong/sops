🎯 1. Purpose & Scope

Manages a structured five-phase AI video production workflow: gather requirements, define global style and characters, plan clips, generate reference images, then execute keyframes, video clips, and audio. Produces professional-grade AI video content for commercials, short films, or any video format.

ℹ️
Scope: In scope: AI-generated video content in 16:9 or 9:16 aspect ratio with optional narration and background music. Out of scope: live-action footage, licensed music, or on-screen dialogue generated via TTS (the video model handles lip-synced audio).

💡 2. When to Use This Skill

💬 Trigger Phrases

Use this skill when Kevin or the team says any of the following:

  • "Create a video" / "Generate a short film"
  • "Make a commercial for [product]"
  • "Produce a video clip about [topic]"
⚠️
Do not use this skill when: Simple image generation — use generate_image directly. This skill is for multi-clip video production only.

Phase 1 — Gather Requirements (STOP)

approval gate
  1. Ask questions across all five dimensions: Strategy, Narrative, Visual Style, Shot Execution, Sound Design.
  2. MUST get explicit user confirmation before proceeding. Never assume or guess missing details.
2

Phase 2 — Global Definitions

automated
  1. Define visual style, character appearances, voice descriptions, and background music mood.
  2. Text-only in this phase — no image generation yet.
3

Phase 3 — Clip Planning

automated
  1. Segment the video into individual clips with planned shots.
  2. Determine which clips need reference images for consistency.
  3. Video prompts must include detailed transition_description (2–4 sentences).
4

Phase 4 — Reference Images (MANDATORY)

automated
  1. Generate reference images before any video keyframes.
  2. NEVER skip this phase. Keyframes without reference images produce inconsistent results.
  3. ALL images must use 16:9 or 9:16 — never 1:1 or other ratios.
5

Phase 5 — Execution

automated
  1. Generate keyframes: last keyframe must show interpolatable change from first (position, state, or composition — not just lighting).
  2. Generate video clips from keyframes.
  3. Narration: generate clip-by-clip, not all at once. Never use TTS for on-screen dialogue or singing.
  4. Audio mixing: overlay all tracks (video audio + narration + BGM). Never replace one with another. Narration must be clearly audible and consistent across clips.

🚨 8. Error Handling

  • Keyframe shows no motion change: Adjust so last keyframe has a clear positional, state, or composition change from the first. Subtle-only changes (lighting/background) cause unnatural video.
  • Aspect ratio wrong: Always use 16:9 or 9:16. Reject any 1:1 or portrait/landscape mix.
  • On-screen dialogue via TTS requested: Do not use TTS for this. The video model handles lip-synced dialogue.

🛠️ 9. Troubleshooting

❌ Video motion looks unnatural or static
✅ Last keyframe must show a meaningful change (subject position, open/close, appear/disappear, composition shift) not just lighting differences.
❌ Audio tracks replacing each other instead of mixing
✅ Overlay all audio tracks. BGM, narration, and video audio must all be present in the final mix.
