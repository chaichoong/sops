🎯 1. Purpose & Scope

Provides a seamless three-phase workflow for onboarding new tenants. Phase 1 creates the tenant record; Phase 2 creates the linked tenancy record with calculated rent dates; Phase 3 updates the rental unit to 'Occupied'. Ensures data consistency across the Tenants, Tenancies, and Rental Units tables in a single session.

ℹ️
Scope: In scope: creating one tenant and one tenancy in a single flow. Out of scope: document generation (use tenant-doc-generator), e-signing (use adobe-sign-field-setup), or post-tenancy comms.

💡 2. When to Use This Skill

💬 Trigger Phrases

Use this skill when Kevin or the team says any of the following:

  • "Onboard a new tenant"
  • "Set up a new tenancy"
  • "Add [name] as a tenant at [property]"
⚠️
Do not use this skill when: The tenant already exists in Airtable — use airtable-tenancy-creator directly, passing the existing tenant Record ID.

1

Phase 1 — Create Tenant Record

automated
  1. Create tenant in Tenants table (tblX4elTuu01gwBYh).
  2. Set Tenant Status → Active, Rent Payment Type, Due Date of the Month.
  3. Save the returned Record ID for Phase 2.
  4. Do NOT proceed to Phase 2 if Phase 1 fails.
2

Phase 2 — Create Tenancy Record

automated
  1. Link Customers field to the tenant Record ID from Phase 1.
  2. Inherit Due Day of Month and Rent Payment Type from Phase 1 inputs.
  3. Calculate Initial Rent Due Date: if today < due day → this month; if today ≥ due day → next month.
  4. Set Payment Frequency → Monthly, Payment Status (Unified) → CFV Actioned.
  5. Do NOT proceed to Phase 3 if Phase 2 fails.
3

Phase 3 — Update Rental Unit to Occupied

automated
  1. Update the Rental Unit record: set Unit Status → Occupied.
  2. If unit is already Occupied: skip this step and note it in the summary.
  3. If Rent Payment Type is Universal Credit: flag clearly in the summary.

🚨 8. Error Handling

  • Phase 1 fails: Do not proceed. Resolve tenant record issue first.
  • Phase 2 fails: Do not proceed to Phase 3. Fix tenancy record issue before updating the unit.
  • Rental Unit not found: Search Rental Units table by name to get the correct linked record ID.

🛠️ 9. Troubleshooting

❌ Existing tenant being reused (skipping Phase 1)
✅ Retrieve the tenant's Due Date of Month and Rent Payment Type from their existing record before Phase 2.
❌ Unit already showing Occupied
✅ Skip Phase 3 and note it in the summary. Check if there is an active tenancy that needs to be ended first.
