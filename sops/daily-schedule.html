🎯 1. Purpose & Scope

This skill builds a practical, realistic daily schedule from a list of tasks and constraints, then adds each block to Google Calendar after Kevin confirms the plan. It accounts for fixed appointments, travel time, multiple locations, and task sequencing logic so the day is achievable — not optimistic.

The goal is a tight, time-honest plan that protects high-priority work and clearly flags anything that cannot fit.

ℹ️
Scope: In scope: planning a single day (today or tomorrow), sequencing flexible tasks, handling travel blocks, adding events to Google Calendar. Out of scope: multi-day planning beyond tomorrow, creating recurring events, sending invitations to other attendees, or modifying existing recurring series.

💡 2. When to Use This Skill

💬 Trigger Phrases

Use this skill when Kevin or the team says any of the following:

  • "Plan my day" / "Can you plan my day?"
  • "Help me schedule" / "Sort my schedule"
  • "What should I do today?"
  • "Schedule tasks for today / tomorrow"
  • "I need to be in [place] by [time] — fit this around it"
  • "I'm leaving at [time] — plan my day around that"
  • "Here are my tasks for the day — put them in order"
  • Any message that includes a task list with durations and a departure or appointment time
⚠️
Do not use this skill when: Kevin asks about task management in Airtable, wants to create standalone calendar events without a full day plan, or is asking about recurring scheduling automation. This skill is for single-day planning only.

1

Gather Constraints

manual
  1. Read Kevin's message and extract any task list, fixed anchors, departure times, and travel information already provided.
  2. Identify what is missing. If the task list is absent, ask for it. If travel is involved but no journey time is given, ask — never guess travel times.
  3. Ask only one question at a time if clarification is needed.
  4. If all required inputs are present in the first message, proceed immediately without asking anything.
ℹ️
Always assume Kevin is starting from home unless he says otherwise. Only include travel blocks when Kevin has provided destination and journey time.
2

Check Current Time

automated
  1. Run a live time check immediately before building the schedule. Use Europe/London timezone.
  2. The schedule start time is always the current time right now — never earlier, never a rounded assumption from earlier in the conversation.
  3. Never build a schedule that begins in the past.
3

Do the Time Arithmetic

automated
  1. Identify available time windows at each location (e.g., arrival to departure).
  2. Subtract fixed anchors from those windows.
  3. Add up all flexible task durations.
  4. Compare total available time against total task time and calculate the shortfall, if any.
  5. Be honest about shortfalls — flag the exact gap. Do not quietly trim tasks; always state what was dropped and why.
4

Sequence the Tasks

automated
  1. Physical or manual tasks (property visits, errands) — schedule earlier before energy drops.
  2. Financial and admin tasks requiring focus — mid-morning when fresh.
  3. Creative or strategic work — after physical demands are out of the way.
  4. Calls and quick check-ins — batch into travel slots where possible.
  5. Tasks with external dependencies — sequence after the dependency is likely resolved.
  6. If Kevin has given a preferred order, respect it. Only suggest a reorder if there is a clear practical reason, and flag it as a suggestion.
5

Present Schedule for Confirmation

approval gate
  1. Present the schedule as a clean table: Time | Task | Duration.
  2. Label travel blocks clearly (e.g. "Travel to Haverhill — calls en route").
  3. Mark fixed anchors clearly.
  4. Include return travel if relevant.
  5. List any tasks that did not fit at the bottom, with a note on why.
  6. Flag if any slots are tight and likely to overrun.
  7. Ask Kevin to confirm before adding anything to the calendar.
⚠️
Approval Gate: Kevin must confirm the schedule before any Google Calendar events are created. Do not write to the calendar without explicit confirmation.
6

Add to Google Calendar

automated
  1. Use gcal_create_event to add every confirmed block to Google Calendar.
  2. Travel blocks are single events (e.g. "Travel to Haverhill — calls en route").
  3. Include a brief description for any event that has context worth remembering (amounts, who to call, what to check).
  4. For meetings with agendas, add the full agenda to the event description.
  5. Confirm the correct date — check with Kevin if there is any ambiguity about which day.
  6. Do not duplicate existing recurring events — check first and update the description instead if one already exists for that slot.
  7. After adding events, check for conflicts with existing calendar entries and flag any overlaps to Kevin.

🚨 8. Error Handling

  • Missing travel time: Never guess. Ask Kevin for the journey duration before proceeding.
  • Tasks exceed available time: Flag the exact shortfall in minutes. State which tasks were dropped or trimmed and why. Do not silently adjust the plan.
  • Calendar write fails: Report the failed event to Kevin, state the error, and ask whether to retry or skip.
  • Conflict detected: Flag the conflict clearly after writing to the calendar. Do not silently overwrite existing events.
  • Ambiguous date: If it is unclear whether the schedule is for today or tomorrow, ask before writing to the calendar.
  • Schedule built in the past: Always re-check the current UK time immediately before presenting the schedule. If the calculated start time is already past, rebuild from the current moment.

🛠️ 9. Troubleshooting

❌ Schedule starts at the wrong time (too early or in the past)
✅ Run a fresh time check in Europe/London timezone immediately before building the schedule. The start time must always be the current real time, not an assumption from earlier in the conversation.
❌ Travel time included but Kevin didn't mention where he's going
✅ Only include travel blocks when Kevin has provided both a destination and a journey duration. Never invent or assume either.
❌ Calendar has duplicate events after the skill runs
✅ Before creating any event, check for existing calendar entries in that time slot. For recurring meetings, update the description on the existing event rather than creating a new one.
❌ Tasks quietly dropped without Kevin knowing
✅ Always list dropped or trimmed tasks at the bottom of the schedule table with the reason. Never silently remove anything from the plan.
❌ Google Calendar tool returns an error during event creation
✅ Report the specific event that failed, state the error message, and ask Kevin whether to retry or skip that event. Do not stop the entire flow for one failed event.
