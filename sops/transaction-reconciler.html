🎯 1. Purpose & Scope

This skill reconciles unreconciled bank transactions in the Operations Director Airtable base by matching them against a history of 7,400+ previously reconciled transactions. It uses Account Alias + Vendor matching with confidence tiers, infers linked records (Business, Cost, Property, Unit, Tenancy, Team Member) where historical precedent or a Learned Rule applies, and presents a structured approval dashboard before writing anything to Airtable.

The skill is self-improving: every run that results in Kevin's amendments must update the Learned Rules section of the SKILL.md, so the same mistake never happens twice. The objective is zero amendments per run.

ℹ️
Scope: In scope — matching, categorising, and linking unreconciled transactions; generating an HTML approval dashboard; writing approved records to Airtable; posting a Slack summary; updating the SKILL.md with learned rules. Out of scope — creating new bank transactions, modifying reconciled records, or reconciling transactions from outside the Operations Director base.

💡 2. When to Use This Skill

Load Context (Phase 1)

automated
  1. Use list_records_for_table with structured filters parameter — NOT filterByFormula (silently ignored).
  2. Filter on fldxKX1IbIFcAOnn5 = false (Reconciled = false) to pull only unreconciled transactions.
  3. Always pass fieldIds and pageSize (e.g. 200) in every request.
  4. Do NOT pull all 7,400+ reconciled records at once — query by exact vendor + account alias for historical matching.
⚠️
Critical: filterByFormula is silently ignored by the Airtable MCP. Always use structured filters.
2

Vendor Matching (Phase 2)

automated

Match each unreconciled transaction against historical reconciled records using Account Alias + Vendor. Apply confidence tiers:

  1. High confidence — 3+ consistent matches: suggest Category, Sub Category, AND every linked field that ALL matches agree on (assessed independently per field).
  2. Medium confidence — mixed results: use mode value with [Mixed History] prefix.
  3. Low confidence — 1–2 matches: use available data with [Low Volume] prefix.
  4. AI Inferred — no vendor match but inference is sensible: use [AI Inferred] prefix.
  5. No Match — flag as [No Match] for Kevin to handle manually.
ℹ️
Business is always inferred from Category — never inherited from the bank account. Learned Rules (Phase 2b) always override generic AI inference.
3

Apply Learned Rules (Phase 2b)

automated

Before generating the dashboard, check every transaction against the Learned Rules. These override generic matching:

  1. Mortgage payments — cross-reference loan reference number to Cost record, always populate Property.
  2. TOGETHER COMMERCIAL FINANCE, BHAM MIDSHIRES, TMW DDR, Secure Trust Bank — link specific Cost records by reference.
  3. COLLINS S + "AIGBURTH" reference — Rental Income, link Aigburth property and active Tenancy/Unit.
  4. Transfer transactions (AmEx, TNT MANAGEMENT, KEVIN BRITTAIN) — Category AND Sub Category both "Transfers".
  5. NOTEMACHINE / ATM withdrawals on Santander — COGS Reactive Maintenance, 22 Newton Street.
  6. Gary Marsh payments — Capital Expenditure; parse property from name; link Property only, no Team Member, no Unit.
  7. Paul Brittain inflows — Loan Receipt / Loan Receipt.
  8. JOO EE BRITTAIN LOAN references — look up matching Loan Repayment cost by date reference.
  9. SHELLEY CO — Operating Expenses / Professional Fees / Real Estate.
  10. Monese CB account — ALL transactions are Personal / Personal Expense Not Tax Deductible. Sub Category by vendor type.
  11. AmEx interest, credit for interest, and payment received — no linked Cost record.
  12. WILLINGHAM WOLVES — Personal / Personal Expense Not Tax Deductible / Discretionary Lifestyle.

Generate Approval Dashboard (Phase 5)

automated
  1. Render reconciliation-YYYY-MM-DD.html to outputs folder. Dark theme. Max width 1500px.
  2. Every linked-record field MUST be a dedicated, always-visible column — never hidden in Notes, tooltips, or a combined "Links" column.
  3. Column order (non-negotiable): Date | Account | Amount | Vendor/Name | Category | Sub Category | Business | Cost | Property | Rental Unit | Tenancy | Tenant | Team Member | Confidence | Notes
  4. Empty linked cells render — in muted colour. Notes column contains rationale only.
  5. Include a Save SKILL.md to clipboard button (top right) on every dashboard — copies the latest skill source for Kevin to paste back to file.
🚫
Non-negotiable: Linked records must NEVER be hidden, collapsed, merged into a single column, or placed in Notes. Any deviation from the mandatory column layout is a defect.
5

Kevin Reviews & Approves (Approval Gate)

approval gate
  1. Present the dashboard HTML file to Kevin.
  2. Kevin reviews each row, amending Category, Sub Category, linked records, or confidence as needed.
  3. Kevin confirms approval — either "Approve all" or item-by-item corrections.
  4. If amendments are made, proceed to Phase 8 (Self-Improvement) before Phase 6 writes.
6

Apply Writes to Airtable (Phase 6)

automated
  1. Resolve Category and Sub Category record IDs by querying the Chart of Accounts - Categories table (tbleWb8ioptnEwPR8) and Sub Categories table (tblOTdRcPf8AgRz25).
  2. Write Category, Sub Category and every approved linked record via update_records_for_table in batches of 10.
  3. Linked records must be written as plain string arrays, e.g. ["recXXX"].
  4. Tick Reconciled field; clear Reconcile Requested field.
  5. Confirm all writes completed successfully before proceeding to Slack.
7

Self-Improvement (Phase 8)

conditional

This phase is MANDATORY after every run where Kevin made amendments. It runs before Phase 6 writes.

  1. Capture every amendment as a permanent Learned Rule in the SKILL.md.
  2. Rebuild the dashboard with all corrections applied.
  3. Re-render the dashboard with the updated SKILL.md loaded into the Save SKILL.md button.
  4. Tell Kevin to click Save SKILL.md and paste the content into the skill file.
  5. Only then proceed with Phase 6 Airtable writes.
✅
Every amendment is a skill defect. The objective is zero amendments. Each Learned Rule prevents the same mistake from recurring.
8

Post Slack Summary (Phase 7)

automated
  1. Post summary to #reconciliation ONLY after Phase 6 writes complete successfully.
  2. Include: total transactions reconciled, breakdown by category/business, any [No Match] items needing manual review, and count of amendments made.

🚨 8. Error Handling

  • Airtable filter returns no results — confirm the filter uses structured filters not filterByFormula. If correct and no unreconciled records exist, report "Nothing to reconcile today" to Kevin.
  • Chart of Accounts ID not found — do not guess. Flag the specific Category or Sub Category that failed to resolve and ask Kevin to confirm the correct value before writing.
  • Linked record write fails — confirm the record ID format is a plain string array ["recXXX"]. Retry in a smaller batch if a batch-level error occurs.
  • Slack post fails — report the failure to Kevin in chat. Do not retry silently. Reconciliation writes are already complete; only the notification failed.
  • [No Match] transactions — surface on the dashboard and flag in the Slack summary. Kevin must handle these manually; do not invent a category.
  • Phase 8 skipped — if amendments were made and Phase 8 was not run, do not apply Phase 6 writes. Stop and complete the self-improvement cycle first.

🛠️ 9. Troubleshooting

❌ Filter returns all records or unexpected results
✅ Confirm you are using the structured filters parameter, not filterByFormula. The MCP silently ignores filterByFormula.
❌ Linked records are showing in Notes column or hidden in expandable panels
✅ This is a dashboard defect. Rebuild the dashboard with each linked-record field as its own dedicated column. The mandatory column layout is non-negotiable.
❌ Mortgage payment not linked to Property
✅ Check the Learned Rule for mortgage interest payments. Cross-reference the loan reference number in the Costs table to find the linked Property record. Apply it.
❌ Monese transaction categorised as "Personal Expense Tax Deductible"
✅ All Monese CB account transactions are "Personal Expense Not Tax Deductible". Update and add the specific vendor as a Learned Rule if it recurs.
❌ Transfer transactions categorised as "Credit Card Payment" or "Intercompany Transfer"
✅ Transfers must use "Transfers" for both Category AND Sub Category. Apply the Learned Rule and capture as a defect in Phase 8.
❌ Gary Marsh payment linked to Team Member or Rental Unit
✅ Gary Marsh is a contractor, not a team member. Only link Property (parsed from the *Name field). Remove Team Member and Unit links.
❌ AmEx interest charge linked to a Cost record
✅ AmEx interest charges, credits, and payment received entries must never have a linked Cost. Leave Costs empty per the Learned Rule.
❌ Phase 8 not triggered after Kevin made amendments
✅ Stop Phase 6 writes immediately. Run Phase 8, capture all amendments as Learned Rules, re-render the dashboard with the Save SKILL.md button, and wait for Kevin to save before proceeding.
