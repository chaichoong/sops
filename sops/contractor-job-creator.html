🎯 1. Purpose & Scope

Adds contractor maintenance jobs to the Tasks table with the Maintenance Ticket checkbox ticked. Generates a concise task name, determines priority (High or Low) from the job description, looks up the property record, and notifies the contractor via Slack. Kevin confirms the summary before the job is created.

ℹ️
Scope: In scope: new maintenance jobs for Gary Marsh, Roy Lavin, or Rob Jackson. Out of scope: general non-maintenance tasks (use airtable-task-creator), marking jobs complete (update Status to Completed).

💡 2. When to Use This Skill

💬 Trigger Phrases

Use this skill when Kevin or the team says any of the following:

  • "Add a job" / "New contractor job" / "Log a maintenance job"
  • "Add to the job list"
  • Kevin describes work that needs doing at a property
  • Kevin mentions a contractor task or repair
⚠️
Do not use this skill when: General admin or team tasks — use airtable-task-creator. Contractor job updates (e.g. marking complete) — search the task and update Status to Completed directly.

AI Processing

automated
  1. Generate Task Name: shorten Kevin's description to a clear, mobile-readable name (e.g. 'Fix boiler - no hot water/heating').
  2. Determine Priority: High = health/safety risks, no heating/hot water, water leaks, structural, security, electrical, gas, fire, flooding, sewage. Low = cosmetic, non-urgent, wear and tear, garden, minor fixtures.
  3. Look up Property: search Properties table (tbl6f0OkAmTC2jbuG) using keywords. Reference references/properties.md for quick matching.
2

Confirm with Kevin

approval gate
  1. Present summary: Task Name, Property (matched address), Contractor, Priority (with brief reasoning), Attachments.
  2. Wait for Kevin's approval or amendments before creating.
3

Phase 1 — Create Task Record

automated
  1. Create with only: Task Name, Assignee (Kevin), Description, and Attachments (if any).
  2. Save returned Record ID for Phase 2.
4

Phase 2 — Update After 30s

automated
  1. Update: Maintenance Ticket → true, Status → Upcoming, Priority level → High/Low Priority, Contractor → contractor name, Properties → [property record ID].
  2. CRITICAL: Set Due Date → null and Time Estimate → null to clear automation defaults.
5

Slack Notification

automated
  1. Post to #property-management (C09EMKREPJL) tagging the contractor by Slack user ID.
  2. Format: 🔧 New job added to your list, @contractor | Task name | Property | Priority | Brief description | Link to job list.

🚨 8. Error Handling

  • Property not found in search: Try alternative keywords (street name, area). Reference references/properties.md. Ask Kevin to clarify if still not found.
  • Phase 2 update fails: Report the Record ID so Kevin can update manually. The Phase 1 record is already created.
  • Slack message fails: Report the failure. Kevin can message the contractor directly.

🛠️ 9. Troubleshooting

❌ Task has a due date after Phase 2
✅ Due Date must be set to null in Phase 2. Check that the update included 'Due Date': null.
❌ Wrong contractor tagged in Slack
✅ Verify Slack user IDs: Gary Marsh → U0A9XD12YPN, Roy Lavin → U0AAN4CTVQQ, Rob Jackson → U0A9MDFKA59.
